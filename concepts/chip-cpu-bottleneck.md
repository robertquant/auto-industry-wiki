# 智驾芯片CPU短板

智驾芯片的CPU算力直接影响规划控制模块性能，是除NPU/GPU之外的另一个关键短板。

## 地平线征程5案例

### 问题表现
- **CPU架构**：8核A55小核
- **CPU算力**：30K DMIPS
- **短板影响**：规划控制模块跑不动，成为系统瓶颈

### 征程6升级
- **CPU架构**：升级到A78AE大核
- **CPU算力**：350K+ DMIPS
- **提升幅度**：10倍+

### 诊断
- J5不是失败产品，是有明确短板的过渡产品
- CPU短板限制了智驾系统整体性能
- 规划控制需要CPU算力，不是NPU算力

## CPU vs NPU/GPU算力

| 模块 | 主要负载 | 关键算力类型 |
|------|----------|--------------|
| 感知 | 图像处理、神经网络推理 | NPU/GPU |
| 预测 | 时序建模、轨迹预测 | NPU |
| 规划控制 | 逻辑计算、约束求解 | CPU |
| 决策 | 状态机、规则引擎 | CPU |

**结论**：感知需要NPU，但规划控制需要CPU。CPU短板会导致系统卡在规划控制环节。

## 行业启示

### 芯片设计教训
- 智驾芯片不能只堆NPU算力，CPU是同等重要的瓶颈
- 过度压成本（使用小核）会在规划控制环节付出代价
- A55小核适合物联网，不适合智驾规划控制

### 竞品对比
- **英伟达Orin**：12核A78AE，CPU算力充足
- **征程6P**：升级A78AE，补齐短板
- **华为MDC**：鲲鹏920系列，CPU算力充沛

### 自研趋势
- 车企自研芯片（比亚迪璇玑、蔚来神玑、小鹏图灵）必须重视CPU算力配置
- 不能重复地平线J5的CPU短板错误
- 端到端架构对CPU需求可能下降，但规划控制仍需强CPU
