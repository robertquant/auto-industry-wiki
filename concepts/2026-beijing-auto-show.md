## 一、舱驾融合 | Cockpit-AD Fusion

### 架构演进

```
传统架构：座舱域控 + 智驾域控（分离，两颗SoC）
    ↓
舱驾融合：单颗SoC同时承载座舱 + ADAS计算
    ↓
中央计算：多域统一，向区域控制器演进
```

### 商业驱动

- **BOM成本降低**：单芯片方案节省15-20%成本
- **系统集成度提升**：减少通信延迟，提升协同效率
- **OTA统一管理**：座舱和智驾软件同步迭代

### 本届车展量产成果

| 方案 | 芯片平台 | 核心能力 | 量产状态 |
|------|----------|----------|----------|
| 高通×哈曼CCU | SA8775 Ride Flex | 单芯片座舱+L2+ ADAS | 已量产（极狐阿尔法T5、S5） |
| 车联天下AL-A1 | 高通骁龙8775 | 全球首个量产单芯片舱驾融合 | 已量产（极狐） |
| 车联天下AL-A2 | 高通骁龙8797 | 下一代中央计算架构 | 预研中 |
| 地平线舱驾一体 | 新发布芯片 | 舱驾一体方案 | 车展首秀 |

### 三强争霸格局

| 阵营 | 芯片平台 | 算力 | 定位 |
|------|----------|------|------|
| 高通 | SA8775 / SA8797 / SA8397 | 单芯片舱驾融合 | 生态最广，60+车企合作 |
| 英伟达 | Thor-U | 700TOPS/颗，双芯1528TOPS | 高算力，智驾导向 |
| 地平线 | 舱驾一体芯片（新发布） | - | 国产替代，性价比 |

### 市场预测（佐思汽研）

- 2026-2030年中国舱驾一体市场CAGR：**36%**
- 2030年市场规模：当前**3.6倍**

### 技术挑战

| 挑战 | 说明 | 解决方案 |
|------|------|----------|
| 算力分配 | 座舱大模型 + 智驾算法共享算力 | QoS优先级调度、分区隔离 |
| 安全隔离 | ADAS功能安全等级（ASIL-D）vs 座舱娱乐 | 虚拟化、内存隔离 |
| 热管理 | 单芯片高功耗 | 液冷方案升级 |
| 算力虚标 | 芯片被降频、功能未开放 | 透明化算力分配，监管介入 |

## 二、智能座舱AI Agent | AI Cockpit Revolution

### 范式转变

| 维度 | 传统语音助手 | AI Agent座舱 |
|------|--------------|--------------|
| 交互模式 | 指令-反馈单向 | 主动感知 + 自主决策 |
| 功能范围 | 单一场景孤立 | 跨端协同 |
| 驱动方式 | 规则引擎 | 大模型驱动 |
| 能力边界 | "能听懂" | "会办事" |

### 代表案例

**宝马 × 阿里千问**
- 三个AI智能体："用车专家"、"出行伙伴"、"百科达人"
- 首搭BMW iX3长轴距版
- 意义：**国际品牌首次采用中国大模型**

**华为鸿蒙座舱小艺智能体**
- 全场景聊天型AI助理
- 自然语言交互 → 自动执行任务链
- 搭载：问界、智界、尊界等

**小米"人车家全生态"**
- AI Agent串联手机、车机、手表、智能家居
- 场景无缝流转

**地平线"咖咖虾"**
- 整车智能体取代传统车载操作系统
- 自然语言控制一切，无需按钮

### 跨端协同案例

**佑驾创新BamBam龙虾助手**：
- 连接车机、电脑、智能家居
- 车内未完成任务 → 到家自动切换到智能音箱继续
- 电池状态 + 路况 + 日程 → 自动规划最优路线和补能

### 本地大模型部署

| 方案 | 模型规模 | 芯片平台 |
|------|----------|----------|
| 华勤HiOS 3.0 | 7B本地部署 | 高通平台 |
| 高通SA8397 | 原生支持端侧大模型 | 骁龙至尊版 |
| 宝马×千问 | 云端协同 | - |

**本地部署优势**：
- 解决延迟问题（实时响应）
- 解决隐私问题（数据不出车）
- 离线可用

## 三、座舱软件架构演进 | Software Architecture

### 软硬解耦 + 模块化

**峰昇座舱软件平台 FusionEX 9.0**：

```
AMCore（应用中间件核心）
    ↓
5大引擎：渲染 + AI + 交互 + 数据 + 安全
    ↓
软硬解耦 + 模块化设计
    ↓
跨终端量产级方案
```

**商业价值**：
- 车企可按需组合功能模块
- 硬件升级不影响上层应用
- OTA频率和效率提升

### AI原生软件开发

**光庭信息SDW 3.0**：
- AI Agent驱动软件工程全生命周期
- 覆盖：架构设计 → 编码 → 测试 → 部署 → 运维
- "自动编程"进入实践阶段

## 四、大模型上车 | LLM On-Vehicle

### 应用层级

| 层级 | 应用场景 | 技术形态 | 代表 |
|------|----------|----------|------|
| L1 | 座舱交互 | 千问、文心、智谱等大模型 | 宝马、华为、小米 |
| L2 | 场景理解 | VLM视觉语言模型 | 小鹏XNGP 4.0 |
| L3 | 驾驶决策 | 端到端大模型 | 理想4D One Model |
| L4 | 云端训练 | WorldEngine、扩散模型 | 华为ADS 5.0 |

### 技术挑战

| 挑战 | 说明 | 解决方案 |
|------|------|----------|
| 推理延迟 | 大模型推理 vs 实时性要求 | 端云协同、模型压缩 |
| 可解释性 | 黑盒决策 vs 安全合规 | 可解释AI探索中 |
| 算力分配 | 座舱大模型 + 智驾算法 | 舱驾融合、双芯片隔离 |
| 数据隐私 | 云端训练 vs 用户隐私 | 本地部署方案 |
